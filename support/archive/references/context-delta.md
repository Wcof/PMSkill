# Context Delta

## 什么是 Context Delta

Context Delta 是每轮 Agent 处理产品上下文后的增量记录。

它的核心目标：把本轮 Agent 处理过程中产生的新事实、新判断、新问题和新影响范围沉淀下来，供下一轮继续使用。

## 为什么需要 Context Delta

1. Agent 对话中的新信息容易丢失。
2. 多轮对话中，前面的结论可能被后面的对话修正。
3. 多 Agent 协作时，需要共享上下文增量。
4. 防止 AI 把推断当成确定事实传递下去。

## 输出位置

```
docs/prd-helper/05-check/context-delta.md
```

## 使用规则

1. 每次执行 prd-helper Skill 必须输出 Context Delta。
2. Context Delta 必须放在 `05-check/` 目录下。
3. 如果有多轮处理，每轮的 Context Delta 可以追加到同一文件或创建新文件。
4. 下一轮处理时，应先读取上一轮的 Context Delta 作为输入。
