# PRD Helper 运行产物

## 中文说明

这是 PRD Helper 的项目运行产物目录。

PRD Helper 按四个业务模块处理产品上下文：

| 模块 | 目录 | 职责 |
|------|------|------|
| 01 Collect 采集 | `01-collect/` | 收集原始材料、建立索引、记录状态 |
| 02 Refine 精炼 | `02-refine/` | 提取事实、决策、约束、问题、假设 |
| 03 Relate 关联 | `03-relate/` | 建立事实、页面、功能、规则、数据、验收之间的关系 |
| 04 Generate 生成 | `04-generate/` | 输出项目说明、页面说明、规则说明、Agent 上下文 |

检查（Check）是贯穿所有模块的质量门禁，不是第五个业务模块。

正常执行顺序：`采集 → 精炼 → 关联 → 生成`，每一步完成后都必须检查。

### 使用方式

- 在 Agent 中发送 `/prd-start` 开启主动采集
- 主动材料由 Agent 自动写入 `01-collect/active/`
- 被动材料（会议纪要、旧 PRD、客户反馈等）直接放入 `01-collect/passive/`
- 采集完成后发送 `/prd-stop`，然后按顺序推进后续模块

### 目录维护

- `01-collect/active/` 由 Agent 和脚本维护，用户不要手动修改
- `01-collect/passive/` 是用户投放区，Agent 不改写其中的原始文件
- `02-refine/` 到 `04-generate/` 由 Agent 在各模块执行时生成
- `05-check/` 存放最终检查结果

## English

This is the PRD Helper project output directory.

PRD Helper processes product context through four business modules:

| Module | Directory | Responsibility |
|--------|-----------|---------------|
| 01 Collect | `01-collect/` | Collect raw materials, build indexes, record state |
| 02 Refine | `02-refine/` | Extract facts, decisions, constraints, questions, assumptions |
| 03 Relate | `03-relate/` | Build relations between facts, pages, features, rules, data, acceptance |
| 04 Generate | `04-generate/` | Output project overview, page specs, rule specs, agent context |

Check is a quality gate across all modules, not a fifth business module.

Execution order: `Collect → Refine → Relate → Generate`, with checks after each step.

### Usage

- Send `/prd-start` in Agent to start active capture
- Active materials are written to `01-collect/active/` by Agent
- Passive materials (meeting notes, legacy PRDs, customer feedback, etc.) go to `01-collect/passive/`
- Send `/prd-stop` when capture is complete, then proceed through subsequent modules

### Directory Maintenance

- `01-collect/active/` is maintained by Agent and scripts — do not edit manually
- `01-collect/passive/` is a user drop zone — Agent does not rewrite original files
- `02-refine/` through `04-generate/` are generated by Agent during module execution
- `05-check/` contains final check results
