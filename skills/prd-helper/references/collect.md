# 采集环节（Collect）

## 目标

采集环节负责保存原始材料。

它不负责判断对错，不负责重写结论，不负责生成最终 PRD。

核心目标：保留需求产生过程中的原始上下文，避免源头信息丢失。

## 输入内容

采集环节支持以下输入：

1. 产品经理口述
2. 会议纪要
3. 聊天记录
4. 客户反馈
5. 评审意见
6. 内部讨论结论
7. 原型说明
8. 页面截图描述
9. Agent 对话摘要
10. 历史 PRD
11. 历史方案
12. 旧系统说明
13. 研发补充说明
14. 测试反馈
15. 变更说明

## 输出目录

```
docs/prd-helper/01-collect/
```

## 输出文件

- 每份材料一个独立文件
- `check.md` 采集检查结果

## 行为规范

1. 不判断、不总结、不直接生成 PRD。
2. 只负责把信息留下来，并按统一结构归档。
3. 保留原始含义，不提前改写成结论。
4. 每份材料必须有来源、类型、时间、涉及模块。
5. 不能把 AI 推断混入原始材料。

## 模板

使用 `01-collect-source-template.md` 创建每份材料文件。

使用 `01-collect-check-template.md` 创建采集检查文件。

## 验收标准

采集环节完成必须满足：

1. 原始材料已保存。
2. 每份材料有来源。
3. 每份材料有类型。
4. 每份材料有涉及模块。
5. 没有提前变成结论。
6. 没有把 AI 推断混入原始材料。
7. 生成 `check.md`。
