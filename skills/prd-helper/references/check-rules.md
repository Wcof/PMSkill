# Check Rules

## Overview

检查是贯穿采集、精炼、关联、生成四个环节的质量门禁。每一步都必须输出 `check.md`。

---

## 采集检查（Collect Check）

输出文件：`docs/prd-helper/01-collect/check.md`

### 检查项

| # | 检查内容 | 通过条件 |
|---|---------|---------|
| 1 | 是否保留原始含义 | 材料内容未被改写为结论 |
| 2 | 是否标记来源 | 每份材料有明确来源 |
| 3 | 是否标记材料类型 | 每份材料有类型（口述/纪要/聊天等） |
| 4 | 是否标记时间或版本 | 每份材料有记录时间 |
| 5 | 是否标记涉及模块 | 每份材料标注了相关产品模块 |
| 6 | 是否没有提前改写成结论 | 材料保持原始表述 |

### 检查模板

```markdown

## 检查时间

## 检查结果

| 检查项 | 结果 | 说明 |
|--------|------|------|
| 保留原始含义 | ✅/❌ | |
| 标记来源 | ✅/❌ | |
| 标记材料类型 | ✅/❌ | |
| 标记时间 | ✅/❌ | |
| 标记涉及模块 | ✅/❌ | |
| 未提前改写 | ✅/❌ | |

## 问题清单

## 建议
```

## 精炼检查（Refine Check）

输出文件：`docs/prd-helper/02-refine/check.md`

### 检查项

| # | 检查内容 | 通过条件 |
|---|---------|---------|
| 1 | 是否区分事实和推断 | facts.md 和 assumptions.md 分开 |
| 2 | 是否提取业务背景 | background.md 非空 |
| 3 | 是否提取业务目标 | goals.md 非空 |
| 4 | 是否提取业务约束 | constraints.md 非空 |
| 5 | 是否提取设计决策 | decisions.md 非空 |
| 6 | 是否标记冲突点 | conflicts.md 记录了冲突或标记为无 |
| 7 | 是否标记待确认问题 | questions.md 记录了问题或标记为无 |
| 8 | 是否每条关键内容都有来源 | 每条事实/决策/约束有来源字段 |

## 关联检查（Relate Check）

输出文件：`docs/prd-helper/03-relate/check.md`

### 检查项

| # | 检查内容 | 通过条件 |
|---|---------|---------|
| 1 | 需求事实是否有关联页面 | 每个 fact 关联到至少一个页面 |
| 2 | 功能是否有关联规则 | 每个 feature 关联到至少一个规则 |
| 3 | 规则是否有关联数据对象 | 每个 rule 关联到至少一个数据对象 |
| 4 | 页面是否有关联验收标准 | 每个 page 关联到至少一个验收项 |
| 5 | 待确认问题是否有关联影响范围 | 每个 question 有影响范围 |
| 6 | 是否存在孤立需求 | 无未关联的 fact |
| 7 | 是否存在孤立页面 | 无未关联的 page |
| 8 | 是否存在孤立规则 | 无未关联的 rule |

| 页面关联验收 | ✅/❌ | |
| 问题关联影响 | ✅/❌ | |

## 生成检查（Generate Check）

输出文件：`docs/prd-helper/04-generate/check.md`

### 检查项

| # | 检查内容 | 通过条件 |
|---|---------|---------|
| 1 | 文档是否基于 refined 和 relations 生成 | 内容可追溯到精炼和关联结果 |
| 2 | 是否没有凭空新增业务规则 | 所有规则有来源 |
| 3 | 是否保留待确认问题 | 待确认问题被标记而非删除 |
| 4 | 是否每个页面有目标、区域、字段、操作、状态 | 页面说明完整 |
| 5 | 是否每个规则有条件、步骤、异常、权限 | 规则说明完整 |
| 6 | 是否每个数据对象有字段语义和关系 | 数据说明完整 |
| 7 | 是否每个核心功能有验收标准 | 验收标准覆盖核心功能 |
| 8 | 是否生成 Agent 可执行上下文 | agent-context 目录有内容 |

### 检查模板

```markdown
# 生成检查报告

## 检查时间

## 检查结果

| 检查项 | 结果 | 说明 |
|--------|------|------|
| 基于 refined 生成 | ✅/❌ | |
| 无凭空新增规则 | ✅/❌ | |
| 保留待确认问题 | ✅/❌ | |
| 页面说明完整 | ✅/❌ | |
| 规则说明完整 | ✅/❌ | |
| 数据说明完整 | ✅/❌ | |
| 验收标准覆盖 | ✅/❌ | |
| Agent 上下文存在 | ✅/❌ | |

## 问题清单

## 建议
```

## 最终总检查（Final Check）

输出目录：`docs/prd-helper/05-check/`

### 输出文件

| 文件 | 用途 |
|------|------|
| `full-check.md` | 整体完整性检查 |
| `gap-check.md` | 缺口分析 |
| `relation-check.md` | 关系完整性检查 |
| `source-check.md` | 来源追溯检查 |
| `generated-check.md` | 生成内容检查 |
| `context-delta.md` | 本轮上下文增量 |
