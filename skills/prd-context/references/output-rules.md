# Output Rules

## 总体原则

1. 所有输出使用 Markdown 格式。
2. 所有输出目录为 `docs/prd-context/`。
3. 每个模块必须输出 `check.md`。
4. 所有关键内容必须有来源追溯。
5. AI 推断必须与事实分开标记。

## 文件命名规范

- 使用小写字母和连字符。
- 使用有意义的名称。
- 示例：`facts.md`、`page-map.md`、`project-overview.md`。

## 目录结构规范

```
docs/prd-context/
├── 01-sources/          # 采集输出
├── 02-refined/          # 精炼输出
├── 03-relations/        # 关联输出
├── 04-generated/        # 生成输出
│   ├── overview/
│   ├── pages/
│   ├── rules/
│   ├── data/
│   ├── acceptance/
│   └── agent-context/
└── 05-check/            # 检查输出
```

## 来源标注规范

每条关键内容必须包含来源标注：

```markdown
- [fact-001] 用户需要支持多租户隔离
  - 来源：01-sources/meeting-notes-20240115.md
  - 可信度：高
  - 状态：已确认
```

## 状态标注规范

所有条目必须有状态标注：

| 状态 | 含义 |
|------|------|
| 已确认 | 经过确认的最终结论 |
| 待确认 | 需要进一步确认 |
| 已否决 | 被明确否决 |
| 已过时 | 被新的决策替代 |

## 可信度标注规范

AI 推断和不确定信息必须标注可信度：

| 可信度 | 含义 |
|--------|------|
| 高 | 有明确来源支持 |
| 中 | 基于多个来源推断 |
| 低 | 仅基于 AI 推断，需人工确认 |

## Agent 上下文输出规范

Agent 上下文必须精简、可执行，包含：

1. 任务背景（简短）
2. 相关页面和规则
3. 关键约束
4. 不可推断内容
5. 待确认问题
6. 验收标准

不同角色的上下文侧重不同：

| 角色 | 侧重 |
|------|------|
| 前端 | 页面说明、字段、交互、状态 |
| 后端 | 业务规则、数据对象、接口用途、权限 |
| 测试 | 验收标准、异常场景、状态流转 |
| 产品 | 项目全貌、决策记录、待确认问题 |
