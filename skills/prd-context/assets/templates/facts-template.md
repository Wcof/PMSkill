# 需求事实

> 从原始材料中提取的已确认或待确认的需求事实。

## 事实清单

| fact_id | 事实内容 | 来源 | 可信度 | 影响范围 | 关联对象 | 状态 |
|---------|---------|------|--------|---------|---------|------|
| fact-001 | | | 高/中/低 | | | 已确认/待确认/已否决/已过时 |

## 填写说明

- **fact_id**：唯一标识，格式 `fact-XXX`
- **事实内容**：简洁描述需求事实
- **来源**：引用原始材料文件路径
- **可信度**：高（有明确来源）/ 中（基于多个来源推断）

> 注意：AI 推断不进入 facts.md，应放入 `assumptions.md`。低可信度的条目应标记为待确认或移入 assumptions。
- **影响范围**：该事实影响的模块或功能
- **关联对象**：关联的页面、功能、规则等
- **状态**：已确认 / 待确认 / 已否决 / 已过时
