## 8 维推断产出（PMContext 片段）

```markdown
## 概述
### 问题与目标（P0:用户场景）
- 事实: 续费率 QoQ -12%（← 对话: 用户反馈）
- 事实: NPS 从 32 跌到 18（← 对话: 用户反馈）
- [假设: 用户期望年付选项, 7/10]（推断自 NPS 关联分析 + 竞品已推）
- [待确认] 年付 LTV 是否显著高于月付（PM 需补充财务数据）
- [冲突] 财务说年付影响现金流 vs 运营说年付提升留存

### 现状平替与摩擦力（P1）
- 事实: 用户手动记录到期日（← 用户访谈 #3）
- [假设: 摩擦点主要是提醒缺失, 6/10]（推断自无提醒模块代码扫描）

### 价值验证度量（P2）
- 事实: 续费率基线 58%（← 对话）
- [假设: 目标续费率 75%, 7/10]（推断自历史最高值）
- [待确认] 年付占比目标（需 PM 定）

## 会员权益
### 事实
- 事实: 当前仅月付，权益=广告去除+高清播放（← 知识库: 会员规则.md）

### 规则（P0:边界条件）
- 事实: 会员三级 标准/高级/至尊（← 知识库）
- [冲突] 权益是否含线下活动——销售 vs 运营分歧（← 对话）
- [假设: 权益变更 24h 同步, 8/10]（推断自通用实践）

### 验收（P0:冲突检测已应用）
- [待确认] 线下活动权益验收（冲突未解，暂不写）

## 全局约束（P2:技术约束）
- 事实: 支付通道 微信+支付宝（← 项目扫描: pay/）
- 事实: MySQL 8.0, 表 users_membership（← 项目扫描）

## 假设清单与验证计划
| 假设 | 置信度 | 验证方式 | 阈值 | 时机 |
|------|--------|---------|------|------|
| 年付提升续费率 | 7/10 | A/B 测试 | +10% | 上线 60 天 |
| 摩擦点=提醒缺失 | 6/10 | 用户访谈验证 | 70%认同 | 上线前 |

## 信息缺口
- 优先级: 年付定价需财务成本模型
- 度量: 缺续费率历史月度数据
- 冲突: 线下活动权益需 PM 决策
```

## 置信度标注决策树

```
材料中有明确依据？
├─ 是 → 事实，标来源
└─ 否 → 可合理推断？
        ├─ 是 → 置信度 ≥ 8？
        │       ├─ 是 → 写为事实附置信度
        │       └─ 否 → [假设] 附置信度(<8)
        └─ 否 → [待确认]，入信息缺口
材料间矛盾？→ [冲突]，保留双方+选更可信方附理由
```

## 实战提示

- **四态标记是核心**：事实/[假设]/[待确认]/[冲突] 必须显式，不能混为一谈
- **置信度 < 8 一律标 [假设]**：宁可暴露信息缺口，也不把假设伪装成事实
- **8 维度全覆盖**：概述/用户/场景/功能/约束/假设/风险/缺口，缺一不可
- **冲突项要保留双方**：不要单方面裁决，留给 PM 审计时决策
- **P0 三维必须有结论**：用户场景/边界条件/冲突检测是底线，材料不足也须标 [待确认]
- **无来源=假设**：这是追溯纪律的底线，无来源的项自动标 [假设]
