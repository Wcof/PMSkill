# Agent 上下文：后端开发

## 1. 本上下文用途

为后端开发 Agent 提供巡检点管理功能的开发上下文，确保 Agent 理解数据模型、业务规则和接口设计。

## 2. 当前任务背景

开发巡检点管理的后端接口，包括 CRUD 接口、关联关系管理、数据校验。

## 3. 必须遵守的事实

- 巡检点和机器人点位是不同对象，有独立的数据表（fact_001）
- 巡检点必须关联到区域（constraint_001）
- 机器人点位必须关联到巡检点（constraint_002）
- 巡检点名称和编号不能重复（rule_003）

## 4. 不能擅自推断的内容

- 不能假设巡检点命名规则已统一（question_001 待确认）
- 不能假设批量导入的技术方案（question_002 待确认）

## 5. 相关页面

- page_003：巡检点管理（接口消费方）

## 6. 相关功能规则

- rule_003：巡检点创建规则
- rule_004：机器人点位创建规则

## 7. 相关数据对象

- data_003：巡检点（InspectionPoint）
- data_004：机器人点位（RobotPoint）
- data_002：区域（Zone）

## 8. 验收标准

- 接口返回数据结构与前端页面字段对应
- 数据校验规则与业务规则一致
- 关联关系正确维护

## 9. 待确认问题

- question_001：巡检点命名规则是否需要统一
- question_002：批量导入的技术方案

## 10. 本轮输出要求

- 巡检点 CRUD 接口
- 巡检点与机器人点位关联接口
- 数据校验逻辑
- 接口文档
