# Agent 上下文：产品评审

## 1. 本上下文用途

为产品评审 Agent 提供巡检点管理功能的评审上下文，确保 Agent 理解需求来源和待确认问题。

## 2. 当前任务背景

评审巡检点管理功能的需求完整性，确认待确认问题。

## 3. 必须遵守的事实

- 巡检点和机器人点位是不同对象（fact_001）
- 系统实施先有地图再有区域和点位（fact_003）
- 设施设备和设施部件是层级关系（fact_004）
- 任务由计划派生（fact_005）

## 4. 不能擅自推断的内容

- 不能假设作业区/施工区层级已确定（conflict_001 待确认）
- 不能假设巡检点命名规则已统一（question_001 待确认）
- 不能假设批量导入方案已确定（question_002 待确认）

## 5. 相关页面

- page_001：地图管理
- page_003：巡检点管理
- page_004：点位管理
- page_005：设施设备管理
- page_007：任务创建

## 6. 相关功能规则

- data_001：地图（Map）

## 8. 验收标准

- acceptance_001：地图与区域管理验收
- acceptance_002：巡检点管理验收
- acceptance_004：设施设备与部件管理验收
- acceptance_005：任务创建与执行验收

## 9. 待确认问题

- conflict_001：作业区/施工区层级是否需要
- question_001：巡检点命名规则
- question_002：批量导入技术方案
- question_003：实时位置展示技术方案
- question_004：检测线具体定义
