# 功能规则：巡检点创建规则

## 1. 规则目标

规范巡检点的创建流程，确保巡检点数据的完整性和一致性。

## 2. 触发页面

- page_003：巡检点管理

## 3. 触发操作

- 用户点击"新建巡检点"按钮
- 用户提交巡检点表单

## 4. 前置条件

- 用户有巡检点管理权限
- 已存在至少一个区域

## 5. 执行步骤

1. 用户点击"新建巡检点"按钮
2. 系统展示新建表单
3. 用户填写：名称（必填）、编号（必填）、所属区域（必填）
4. 用户提交表单
5. 系统校验：名称是否重复、编号是否重复、区域是否存在
6. 校验通过：创建巡检点，状态为"已创建"
7. 校验失败：展示错误信息

## 6. 状态变化

- 无 → 已创建：巡检点创建成功
- 已创建 → 已启用：巡检点启用
- 已启用 → 已停用：巡检点停用

## 7. 异常分支

- 名称重复：提示"巡检点名称已存在"
- 编号重复：提示"巡检点编号已存在"
- 区域不存在：提示"所选区域不存在"

## 8. 权限规则

- 管理员：可创建、编辑、删除
- 普通用户：可查看，不可创建、编辑、删除

## 9. 幂等要求

- 相同名称和编号的巡检点不能重复创建

## 10. 日志与审计

- 记录创建人、创建时间
- 记录修改人、修改时间
- 记录删除人、删除时间

## 11. 关联数据对象

- data_003：巡检点（InspectionPoint）
- data_002：区域（Zone）

## 12. 建议接口用途

- POST /api/inspection-points：创建巡检点
- PUT /api/inspection-points/:id：更新巡检点
- DELETE /api/inspection-points/:id：删除巡检点
- GET /api/inspection-points：查询巡检点列表

## 13. 验收标准

- 能成功创建巡检点
- 名称和编号不能重复
- 必须关联到区域
- 创建后状态为"已创建"
- 操作日志完整

## 14. 待确认问题
