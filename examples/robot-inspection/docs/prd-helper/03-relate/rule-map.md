# 规则关联图

## rule_001：地图创建规则

- 规则说明：创建地图时必须填写名称和描述，地图创建后可以在其上划分区域
- 来源事实：fact_003
- 触发功能：feature_001
- 触发页面：page_001
- 前置条件：用户有地图管理权限
- 状态变化：无 → 已创建
- 异常场景：地图名称重复
- 关联数据对象：data_001
- 关联验收标准：acceptance_001

## rule_002：区域划分规则

- 规则说明：在地图上划分区域，区域必须属于地图，区域可以标记巡检点
- 来源事实：fact_003
- 触发功能：feature_002
- 触发页面：page_002
- 前置条件：地图已创建
- 状态变化：无 → 已划分
- 异常场景：区域重叠
- 关联数据对象：data_002
- 关联验收标准：acceptance_001

## rule_003：巡检点创建规则

- 规则说明：巡检点必须关联到区域，支持关联多个机器人点位
- 来源事实：fact_001、fact_009
- 触发功能：feature_003
- 触发页面：page_003
- 前置条件：区域已划分
- 状态变化：无 → 已创建
- 异常场景：巡检点名称重复
- 关联数据对象：data_003
- 关联验收标准：acceptance_002

## rule_004：机器人点位创建规则

- 规则说明：机器人点位必须关联到巡检点，支持关联多个采集位
- 来源事实：fact_001、fact_002
- 触发功能：feature_004
- 触发页面：page_004
- 前置条件：巡检点已创建
- 状态变化：无 → 已创建
- 异常场景：点位坐标超出区域范围
- 关联数据对象：data_004
- 关联验收标准：acceptance_003

## rule_005：设施设备创建规则

- 规则说明：设施设备支持手动录入和批量导入，必须填写名称、编号、类型
- 来源事实：fact_004、fact_008
- 触发功能：feature_005
- 触发页面：page_005
- 前置条件：用户有设施设备管理权限
- 状态变化：无 → 已录入
- 异常场景：编号重复、导入格式错误
- 关联数据对象：data_005
- 关联验收标准：acceptance_004

## rule_006：设施部件关联规则

- 规则说明：设施部件必须关联到设施设备，部件可以关联检测线
- 来源事实：fact_004、fact_006
- 触发功能：feature_006
- 触发页面：page_006
- 前置条件：设施设备已创建
- 状态变化：无 → 已关联
- 异常场景：部件编号重复
- 关联数据对象：data_006
- 关联验收标准：acceptance_004

## rule_007：任务执行规则

- 规则说明：任务从计划派生或手动创建，任务执行链路为：计划→派生→执行→到达→采集→检测→记录。支持中断和恢复。
- 来源事实：fact_005、fact_007
- 触发功能：feature_007
- 触发页面：page_007
- 前置条件：巡检计划已创建或手动指定
- 状态变化：待执行 → 执行中 → 已完成 / 已中断
- 异常场景：机器人故障、网络中断
- 关联数据对象：data_007
- 关联验收标准：acceptance_005

## rule_008：计划派生任务规则

- 规则说明：巡检计划按执行周期自动派生任务，派生时自动关联巡检点和检测线
- 来源事实：fact_005
- 触发功能：feature_008
- 触发页面：page_008
- 前置条件：巡检计划已配置
- 状态变化：待派生 → 已派生
- 异常场景：计划配置错误、无可用机器人
- 关联数据对象：data_008
- 关联验收标准：acceptance_006
