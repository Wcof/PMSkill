# 项目说明：机器人巡检管理系统

## 1. 项目背景

本项目是一个机器人巡检管理系统，用于替代传统的人工巡检方式。系统通过机器人自动执行巡检任务，管理巡检地图、巡检点位、设施设备和巡检任务。

核心问题：传统人工巡检效率低、成本高、存在安全风险，需要通过机器人自动化巡检来提升效率和安全性。

## 2. 业务目标

1. 建立清晰的巡检点位体系，明确区分巡检点、机器人点位、采集位
2. 提高设施设备管理效率，支持批量导入
3. 实现任务自动派生，减少人工创建
4. 优化检测结果查看体验

## 3. 使用角色

- 产品经理：定义需求和业务规则
- 实施工程师：现场部署和配置系统
- 巡检人员：使用系统查看巡检计划和结果
- 机器人操作员：监控机器人执行状态
- 设施设备管理员：管理设施设备和部件信息

## 4. 系统范围

包含：地图管理、区域划分、巡检点管理、机器人点位管理、设施设备管理、设施部件管理、任务管理、巡检计划管理。

不包含：机器人硬件控制、底层通信协议、视频监控集成。

## 5. 核心模块

1. 地图管理：管理地图和区域
2. 巡检管理：管理巡检点和机器人点位
3. 设施设备管理：管理设施设备和部件
4. 任务管理：管理巡检计划和任务执行

## 6. 核心流程

```
创建地图 → 划分区域 → 标记巡检点 → 配置机器人点位 → 录入设施设备 → 创建巡检计划 → 派生任务 → 执行巡检 → 记录检测结果
```

## 7. 关键业务约束

- 巡检点必须关联到区域
- 机器人点位必须关联到巡检点
- 设施部件必须关联到设施设备
- 任务必须有执行机器人
- 检测结果必须关联到设施设备或部件

## 8. 非目标范围

- 不做机器人硬件控制
- 不做底层通信协议
- 不做视频监控集成
- 不做 SaaS 平台
- 不做复杂多人协作

## 9. 待确认问题

- 作业区/施工区层级是否需要增加
- 巡检点命名规则是否需要统一
- 批量导入的技术方案
- 检测线的具体定义
