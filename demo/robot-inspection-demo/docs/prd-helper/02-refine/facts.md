# 需求事实清单

## 1. 明确事实

### fact_001：巡检点和机器人点位是不同对象

- 事实内容：巡检点是业务人员理解的巡检单元，机器人点位是机器人执行的细粒度点位。一个巡检点可以包含多个机器人点位。
- 来源材料：产品口述、会议纪要
- 来源位置：product-voice.md、meeting-notes.md
- 涉及模块：巡检管理
- 相关页面：巡检点管理、点位管理
- 可信度：高
- 状态：已确认

### fact_002：采集位是机器人点位上的具体检测位置

- 事实内容：机器人到达点位后停车，通过采集位判断并执行检测。停车点、巡检点、采集位是三个不同概念。
- 来源材料：产品口述、Agent 对话摘要
- 来源位置：product-voice.md、agent-session.md
- 涉及模块：巡检管理
- 相关页面：点位管理
- 可信度：高
- 状态：已确认

### fact_003：系统实施先有地图再有区域和点位

- 事实内容：系统实施时，先创建地图，再在地图上划分区域，区域里面标记点位。
- 来源材料：产品口述
- 来源位置：product-voice.md
- 涉及模块：地图管理
- 相关页面：地图管理、地图编辑
- 可信度：高
- 状态：已确认

### fact_004：设施设备和设施部件是层级关系

- 事实内容：设施设备需要录入，设施部件需要关联到设施设备。阀门、仪表应作为独立部件对象。
- 来源材料：产品口述、会议纪要
- 来源位置：product-voice.md、meeting-notes.md
- 涉及模块：设施设备管理
- 相关页面：设施设备管理、设施设备编辑
- 可信度：高
- 状态：已确认

### fact_005：任务由计划派生

- 事实内容：任务执行链路为：计划 → 派生任务 → 任务执行 → 到达点位 → 采集 → 检测 → 记录结果。
- 来源材料：产品口述、会议纪要、Agent 对话摘要
- 来源位置：product-voice.md、meeting-notes.md、agent-session.md
- 涉及模块：任务管理
- 相关页面：巡检计划、任务创建
- 可信度：高
- 状态：已确认

### fact_006：检测线是检测动作的集合

- 事实内容：检测线需要关联到具体的设施设备或部件，检测结果需要记录到对应的设施设备或部件上。
- 来源材料：会议纪要
- 来源位置：meeting-notes.md
- 涉及模块：巡检管理、设施设备管理
- 相关页面：设施设备编辑
- 可信度：高
- 状态：已确认

### fact_007：任务需要支持中断和恢复

- 事实内容：任务执行过程中可能需要中断，中断后需要支持恢复执行。
- 来源材料：会议纪要
- 来源位置：meeting-notes.md
- 涉及模块：任务管理
- 相关页面：任务创建
- 可信度：高
- 状态：已确认

### fact_008：客户需要批量导入设施设备

- 事实内容：目前设施设备需要逐个录入，客户反馈效率低，建议支持从 Excel 批量导入。
- 来源材料：客户反馈
- 来源位置：customer-feedback.md
- 涉及模块：设施设备管理
- 相关页面：设施设备管理
- 可信度：高
- 状态：已确认

### fact_009：客户需要统一巡检点命名规则

- 事实内容：不同区域的巡检点命名规则不一致，建议统一为：区域编号-巡检点编号。
- 来源材料：客户反馈
- 来源位置：customer-feedback.md
- 涉及模块：巡检管理
- 相关页面：巡检点管理
- 可信度：中
- 状态：待确认

### fact_010：客户需要在巡检点页面查看检测结果

- 事实内容：目前需要进入设施设备详情才能查看检测结果，建议在巡检点页面也能查看。
- 来源材料：客户反馈
- 来源位置：customer-feedback.md
- 涉及模块：巡检管理
- 相关页面：巡检点管理
- 可信度：中
- 状态：待确认

### fact_011：Agent 曾混淆巡检点和机器人点位

- 事实内容：Agent 在历史对话中把巡检点和机器人点位理解成同一个对象，需要在文档中明确区分。
- 来源材料：Agent 对话摘要
- 来源位置：agent-session.md
- 涉及模块：巡检管理
- 相关页面：巡检点管理、点位管理
- 可信度：高
- 状态：已确认

### fact_012：机器人是执行工具不是设施设备

- 事实内容：Agent 曾把机器人归入设施设备管理，实际上机器人是执行工具，不是被巡检的设施设备。
- 来源材料：Agent 对话摘要
- 来源位置：agent-session.md
- 涉及模块：设施设备管理、任务管理
- 相关页面：设施设备管理
- 可信度：高
- 状态：已确认
