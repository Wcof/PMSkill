# 业务背景

## 1. 背景摘要

本项目是一个机器人巡检管理系统，用于管理巡检地图、巡检点位、设施设备和巡检任务。系统通过机器人自动执行巡检任务，替代传统的人工巡检方式。

## 2. 业务现状

- 当前已有地图管理功能，支持在地图上标记巡检点
- 设施设备需要手动逐个录入
- 任务由人工创建，不支持从计划自动派生
- 检测结果需要进入设施设备详情才能查看

## 3. 当前痛点

- 巡检点和机器人点位概念混淆，导致系统设计不清晰
- 设施设备录入效率低，不支持批量导入
- 任务创建流程繁琐，不支持从计划自动派生
- 检测结果查看路径过深，操作不便
- Agent 容易理解错业务概念，导致开发返工

## 4. 相关角色

- 产品经理：定义需求和业务规则
- 实施工程师：现场部署和配置系统
- 巡检人员：使用系统查看巡检计划和结果
- 机器人操作员：监控机器人执行状态
- 设施设备管理员：管理设施设备和部件信息
