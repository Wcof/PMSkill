## 关键词

## 原始内容

传统 PRD 主要用于交付、评审和留痕。

Agent Context PRD 主要用于让前端、后端、测试和开发 Agent 快速理解项目上下文。

页面说明主要给前端使用。

业务规则主要给后端理解逻辑、状态和接口意图。

数据说明主要描述业务对象和关系，后端 Agent 再结合现有库表补充技术结构。

Agent 上下文需要精简、可执行，包含：任务背景、相关页面、相关规则、关键约束、不可推断内容、待确认问题、验收标准。

不同角色需要不同的上下文：
- 前端：页面说明、字段、交互、状态
- 后端：业务规则、数据对象、接口用途、权限
- 测试：验收标准、异常场景、状态流转
- 产品：项目全貌、决策记录、待确认问题
