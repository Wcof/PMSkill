## 原始内容

PRD helper 是面向产品经理的产品上下文处理工具。

它不是单纯的 PRD 生成器，而是围绕产品开发过程中上下文容易丢失、信息难以复用、研发和测试反复补充背景的问题，建立从原始沟通到结构化 PRD 的处理链路。

核心链路是：采集 -> 精炼 -> 关联 -> 生成。

PRD helper 的目标不是单纯生成一份传统 PRD，而是帮助产品经理把产品开发过程中的原始沟通、需求判断、原型说明、业务规则、数据对象、验收标准和 Agent 对话上下文沉淀下来。

在 AI 辅助开发过程中，常见问题包括：

1. 产品经理和客户、研发、测试、Agent 的沟通内容容易丢失。
2. 原型只能表达页面结果，不能完整表达业务背景、规则约束和设计取舍。
3. 前端、后端、测试会各自补上下文，导致理解不一致。
4. Claude Code、Codex、Trae、Cursor 等 Agent 对话中产生的新上下文很难沉淀。
5. AI 容易把推断当成确定事实。
6. 多人、多 Agent 协作时，缺少统一的产品上下文。
7. 需求、页面、功能、规则、数据、验收之间容易断链。

因此，第一阶段不应先做完整平台，而应先做一个能验证核心链路的 Skill Demo。
