# 前端 Agent 上下文

## 任务背景

PRD Context Skill Kit 是一个通用 Agent Skill 套件，用于处理产品上下文。前端 Agent 的任务是理解页面说明、字段、交互和状态，实现前端相关功能。

## 相关页面

| 页面名称 | 页面路径 | 关键内容 |
|---------|---------|---------|
| PRD helper 上下文处理流程 | docs/prd-context/ | 采集→精炼→关联→生成→检查的完整流程 |

## 相关规则

| 规则名称 | 规则要点 |
|---------|---------|
| 采集规则 | 保留原始内容、标记来源和类型 |
| 精炼规则 | 区分事实和推断、标记冲突和待确认 |
| 生成规则 | 角色化输出、基于 refined 和 relations |

## 相关数据

| 数据对象 | 关键字段 | 关系 |
|---------|---------|------|
| source | 标题、类型、来源、时间、内容 | 被精炼为其他对象 |
| fact | fact_id、内容、来源、可信度、状态 | 关联到页面/功能/规则 |
| page-spec | 页面名称、路径、目标、区域、字段、操作、状态 | 来自 page-map |

## 关键约束

- 不做网页 UI
- 不做后台系统
- 所有输出为 Markdown 文件

## 不可推断内容

- 不可推断页面的视觉设计
- 不可推断交互的具体实现方式
- 不可推断字段的验证规则

## 待确认问题

- Trae 的实际 Skill 文件落点需要验证（question-001）

## 验收标准

- [ ] 能正确理解页面说明的结构
- [ ] 能区分事实和推断
- [ ] 能识别待确认问题
- [ ] 能基于上下文生成前端代码

## 执行建议

1. 先阅读 04-generated/pages/ 下的页面说明
2. 理解页面的目标、区域、字段、操作、状态
3. 注意待确认问题，不要自行推断
4. 生成代码前先确认关键约束
