# 后端 Agent 上下文

## 任务背景

PRD Context Skill Kit 是一个通用 Agent Skill 套件，用于处理产品上下文。后端 Agent 的任务是理解业务规则、数据对象和接口用途，实现后端相关功能。

## 相关页面

| 页面名称 | 页面路径 | 关键内容 |
|---------|---------|---------|
| PRD helper 上下文处理流程 | docs/prd-context/ | 采集→精炼→关联→生成→检查的完整流程 |

## 相关规则

| 规则名称 | 规则要点 |
|---------|---------|
| 上下文精炼与关联规则 | 从原始材料提取结构化信息并建立关系 |
| 检查规则 | 每步输出 check.md，最终输出 05-check |
| Delta规则 | 输出本轮增量，供下轮使用 |

## 相关数据

| 数据对象 | 关键字段 | 关系 |
|---------|---------|------|
| source | 标题、类型、来源、时间、内容 | 被精炼为其他对象 |
| fact | fact_id、内容、来源、可信度、状态 | 关联到页面/功能/规则 |
| rule-spec | 规则名称、目标、前置条件、步骤、异常、权限 | 来自 rule-map |
| data-spec | 数据域、核心对象、字段语义、状态、生命周期 | 来自 data-map |

## 关键约束

- 不做数据库
- 不做后台系统
- 所有数据存储为 Markdown 文件
- 数据对象之间通过引用关联

## 不可推断内容

- 不可推断数据库表结构
- 不可推断 API 接口设计
- 不可推断数据验证规则

## 待确认问题

- 多轮处理时 Context Delta 如何累积（question-003）

## 验收标准

- [ ] 能正确理解业务规则的结构
- [ ] 能理解数据对象的关系
- [ ] 能识别关键约束
- [ ] 能基于上下文生成后端代码

## 执行建议

1. 先阅读 04-generated/rules/ 下的规则说明
2. 理解规则的前置条件、执行步骤、异常分支
3. 阅读 04-generated/data/ 下的数据说明
4. 理解数据对象的字段语义和关系
5. 注意不可推断内容，不要自行设计数据库结构
