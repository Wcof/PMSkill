# 测试 Agent 上下文

## 任务背景

PRD Context Skill Kit 是一个通用 Agent Skill 套件，用于处理产品上下文。测试 Agent 的任务是理解验收标准、异常场景和状态流转，执行测试相关任务。

## 相关页面

| 页面名称 | 页面路径 | 关键内容 |
|---------|---------|---------|
| PRD helper 上下文处理流程 | docs/prd-context/ | 采集→精炼→关联→生成→检查的完整流程 |

## 相关规则

| 规则名称 | 规则要点 |
|---------|---------|
| 检查规则 | 每步输出 check.md，最终输出 05-check |
| 上下文精炼与关联规则 | 从原始材料提取结构化信息并建立关系 |

## 相关数据

| 数据对象 | 关键字段 | 关系 |
|---------|---------|------|
| acceptance | 功能验收、页面验收、规则验收、异常场景、权限场景 | 来自 acceptance-map |
| check | 检查项、结果、说明 | 每步输出 |

## 关键约束

- 每步必须输出 check.md
- 最终必须输出 05-check/ 下的所有检查文件
- 必须输出 Context Delta

## 不可推断内容

- 不可推断验收标准
- 不可推断异常场景
- 不可推断状态流转

## 待确认问题

- Demo 是否需要包含脚本检查运行结果（question-002）

## 验收标准

- [ ] 能正确理解验收标准
- [ ] 能识别异常场景
- [ ] 能理解状态流转
- [ ] 能基于上下文生成测试用例

## 执行建议

1. 先阅读 04-generated/acceptance/ 下的验收标准
2. 理解功能验收、页面验收、规则验收
3. 关注异常场景和边界条件
4. 检查每个目录的 check.md 是否完整
5. 验证 Context Delta 是否包含所有增量
