# PRD helper 上下文数据说明

## 基本信息

| 字段 | 内容 |
|------|------|
| 数据域 | PRD 上下文 |
| 核心对象 | source, fact, decision, constraint, conflict, question, assumption, relation, context-delta |

## 对象关系

```
source --精炼--> fact
source --精炼--> decision
source --精炼--> constraint
source --精炼--> conflict
source --精炼--> question
source --精炼--> assumption

fact --关联--> page-spec
fact --关联--> feature
feature --关联--> rule-spec
rule-spec --关联--> data-spec
page-spec --关联--> acceptance

assumption --确认--> fact (或 rejection)

context-delta --记录--> 本轮所有变更
```

## 字段语义

## 状态字段

| 对象 | 状态字段 | 可选值 | 含义 |
|------|---------|--------|------|
| fact | 状态 | 已确认/待确认/已否决/已过时 | 事实的确认状态 |
| question | 状态 | 待确认/已确认/已关闭 | 问题的处理状态 |
| assumption | 状态 | 待确认/已确认/已否决 | 推断的确认状态 |
| conflict | 处理状态 | 待处理/已解决/已搁置 | 冲突的处理状态 |
| relation | 确认状态 | 已确认/待确认 | 关联的确认状态 |

## 生命周期

所有对象遵循统一生命周期：

1. **创建**：Agent 从原始材料中提取
2. **待确认**：等待产品经理确认
3. **已确认**：经过确认的最终结论
4. **已否决**：被明确否决
5. **已过时**：被新的决策替代

## 数据来源

所有数据来源于原始材料（01-sources/），通过 Agent 精炼提取。

## 数据去向

精炼后的数据用于：
- 建立关联关系（03-relations/）
- 生成角色化文档（04-generated/）
- 输出 Context Delta（05-check/）
