# PRD helper 上下文处理流程

## 基本信息

| 字段 | 内容 |
|------|------|
| 页面名称 | PRD helper 上下文处理流程 |
| 页面路径 | docs/prd-context/ |
| 页面目标 | 把原始产品沟通转化为结构化、可追溯、可被 Agent 使用的产品上下文 |
| 使用角色 | 产品经理、前端 Agent、后端 Agent、测试 Agent |

## 页面区域

本项目为 Skill Kit，"页面"等价于输出文档目录结构：

| 区域 | 说明 | 输出目录 |
|------|------|---------|
| 采集区 | 保存原始材料 | 01-sources/ |
| 精炼区 | 提取结构化信息 | 02-refined/ |
| 关联区 | 建立对象关系 | 03-relations/ |
| 生成区 | 输出角色化文档 | 04-generated/ |
| 检查区 | 质量门禁和增量记录 | 05-check/ |

## 字段展示

### 采集区字段

| 字段名称 | 字段类型 | 数据来源 | 是否必填 | 说明 |
|---------|---------|---------|---------|------|
| 材料标题 | 文本 | 用户输入 | 是 | 材料的简短标题 |
| 材料类型 | 枚举 | 用户输入 | 是 | 口述/纪要/聊天/反馈/Agent对话等 |
| 来源 | 文本 | 用户输入 | 是 | 材料的来源渠道 |
| 记录时间 | 日期 | 用户输入 | 是 | 材料记录的时间 |
| 原始内容 | 长文本 | 用户输入 | 是 | 保留原始表述 |

### 精炼区字段

| 字段名称 | 字段类型 | 数据来源 | 是否必填 | 说明 |
|---------|---------|---------|---------|------|
| fact_id | 标识 | 系统生成 | 是 | 唯一标识 |
| 事实内容 | 文本 | AI提取 | 是 | 简洁描述需求事实 |
| 来源 | 引用 | AI提取 | 是 | 引用原始材料 |
| 可信度 | 枚举 | AI判断 | 是 | 高/中/低 |
| 状态 | 枚举 | 人工确认 | 是 | 已确认/待确认/已否决/已过时 |

## 按钮操作

| 操作名称 | 操作类型 | 触发条件 | 操作结果 | 权限要求 |
|---------|---------|---------|---------|---------|
| 采集材料 | Agent执行 | 用户提供原始材料 | 生成 01-sources/ 文件 | 无 |
| 精炼材料 | Agent执行 | 采集完成 | 生成 02-refined/ 文件 | 无 |
| 建立关联 | Agent执行 | 精炼完成 | 生成 03-relations/ 文件 | 无 |
| 生成文档 | Agent执行 | 关联完成 | 生成 04-generated/ 文件 | 无 |
| 运行检查 | Agent执行 | 每步完成后 | 生成 check.md | 无 |
| 输出 Delta | Agent执行 | 全部完成后 | 生成 context-delta.md | 无 |

## 交互状态

| 状态 | 触发条件 | 表现 |
|------|---------|------|
| 默认状态 | 无输入 | 提示用户提供原始材料 |
| 采集中 | 用户提供材料 | 保存到 01-sources/ |
| 精炼中 | 采集完成 | 提取结构化信息 |
| 关联中 | 精炼完成 | 建立对象关系 |
| 生成中 | 关联完成 | 输出角色化文档 |
| 检查中 | 每步完成后 | 运行检查规则 |
| 完成 | 全部完成 | 输出 Context Delta |

## 空状态

当 `docs/prd-context/` 目录为空时，提示用户：请提供原始产品材料（会议纪要、聊天记录、原型说明等）。

## 异常状态

| 异常 | 表现 | 处理方式 |
|------|------|---------|
| 材料来源不明 | check.md 标记缺失 | 要求补充来源 |
| 事实与推断混淆 | check.md 标记问题 | 要求区分 |
| 存在孤立对象 | relation check 标记 | 要求补充关联 |

## 权限展示

| 角色 | 可见内容 | 可操作内容 |
|------|---------|-----------|
| 产品经理 | 全部内容 | 全部操作 |
| 前端 Agent | 前端上下文、页面说明 | 执行前端相关任务 |
| 后端 Agent | 后端上下文、规则和数据说明 | 执行后端相关任务 |
| 测试 Agent | 测试上下文、验收标准 | 执行测试相关任务 |

## 关联规则

- 采集规则：保留原始内容、标记来源和类型
- 精炼规则：区分事实和推断、标记冲突和待确认
- 关联规则：建立需求→页面→规则→数据→验收关系
- 生成规则：角色化输出、基于 refined 和 relations
- 检查规则：每步输出 check.md、最终输出 05-check

## 关联数据

- source（原始材料）
- fact（需求事实）
- decision（设计决策）
- constraint（业务约束）
- conflict（冲突点）
- question（待确认问题）
- assumption（AI推断）
- relation（关联关系）
- context-delta（增量记录）
